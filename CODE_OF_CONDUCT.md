# MRI Second Opinion Community Conduct

## Purpose

This repository is intended to be a focused, technical, clinician-reviewed MRI workflow project.

Discussion and collaboration should stay respectful, evidence-based, and useful to contributors and maintainers.

## Expected Behavior

1. be respectful in technical disagreement
2. critique code, claims, and evidence rather than people
3. keep discussions scoped to the repository and its stated goals
4. avoid hype, harassment, intimidation, or discriminatory behavior
5. avoid posting PHI, secrets, or exploit details in public threads

## Unacceptable Behavior

1. harassment, threats, or personal attacks
2. discriminatory or hateful language
3. posting real patient data or other sensitive material
4. publishing exploit details for unpatched vulnerabilities in public issues
5. repeated off-scope promotion, spam, or bad-faith disruption

## Reporting

For behavior that affects contributor safety or repository trust, contact the maintainers through the repository's documented security or maintainer path before escalating publicly.

For active safety or abuse concerns in this repository, use the shared repository inbox `security@microphoenix.io` with the subject prefix `[mri-second-opinion][conduct]` until a repo-specific conduct contact is published.

## Enforcement

Maintainers may remove comments, close discussions, or restrict participation when conduct harms the project or its contributors.

## Scope

This document applies to repository issues, pull requests, code review, and discussion spaces directly associated with MRI Second Opinion.
