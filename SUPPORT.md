# Support

## What this repository supports today

This repository supports public discussion and contribution around the MRI-only second-opinion workflow baseline that is currently implemented.

Use support channels conservatively:

1. bug reports and reproducible runtime problems: GitHub Issues with the provided templates
2. documentation drift or scope confusion: GitHub Issues with links to the affected docs
3. proposed code changes: Pull Requests with updated docs and verification notes

## What is not a support channel

1. real patient data or PHI
2. emergency or clinical decision support
3. private security reports in public issues

## Security

Security reports must follow the process in `SECURITY.md`.

## Scope questions

Before opening an issue, read:

1. `README.md`
2. `docs/scope-lock.md`
3. `docs/api-scope.md`
4. `docs/releases/v1-go-no-go.md`

## Response expectations

This is a conservative public-preparation repository, not a staffed product support desk.

Best-effort triage is the intended model until a dedicated maintainer team is declared.
